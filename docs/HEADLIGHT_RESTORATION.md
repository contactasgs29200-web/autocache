# Headlight Restoration — AI Pipeline

The "Lustrage des optiques" feature reconstructs yellowed / oxidized /
foggy headlight lenses using an image-to-image / inpainting model. The
masked headlight region is regenerated by the AI; the rest of the photo
is left **bit-identical** thanks to client-side compositing.

> Goal: realistic, transparent, glossy plastic-or-glass lenses with
> depth, reflections, and the same shape as the source. **No** "yellow
> pixel → white pixel" replacement, **no** local OpenCV filter on the
> hot path.

---

## Pipeline overview — full-image edit (primary), per-headlight crops (fallback)

We tried per-headlight crops as the primary mode (it stops the model
from redesigning ONE optic differently from the other), but on real
photos the result showed seams, hue mismatches between the AI crop and
the rest of the body, or grey blobs on missed optics. The full-image
ChatGPT-style prompt — "make the headlights less yellow and more
transparent, without changing the rest of the photo" — gave a
noticeably better premium feel, so it's now the primary mode and
per-headlight is a safety net.

```
[1] Photo (front)
       │
       ▼
[2] api/headlights.js          — detects each front headlight (polygon + bbox)
       │
       ▼
[3] FULL-IMAGE attempt 1 (default strict prompt)
    ├─ createFullImageEditAssets(ctx, W, H, lights)
    │     • photo downsampled to 1024² / 1024×1536 / 1536×1024
    │     • mask: opaque everywhere EXCEPT the headlight polygons
    └─ POST /api/lustrage-pro { imageBase64, maskBase64, size, mode:"ai", strength:"restore" }
       │
       ▼
[4] HeadlightRestorationProvider (openai / replicate / stability)
       │
       ▼
[5] validateFullImageResult(beforeCanvas, aiCanvas, lights)
       • mean RGB diff inside  the headlight polygons → must be > 4
       • mean RGB diff outside the headlight polygons → must be < 8
       • % of pixels with strong diff (>40) outside → must be < 3%
       │ ok? ───────────────────────► composite at full canvas, return
       │
       ▼ rejected
[6] FULL-IMAGE attempt 2 (STRICT_RETRY_PROMPT)
       • same image+mask, even stricter prompt
       • re-validate; if ok, composite, return
       │
       ▼ rejected again
[7] PER-HEADLIGHT CROP fallback
       • one independent inpaint per optic on a tight crop
       • per-optic meanDiff check; rollback any optic that didn't change
       │
       ▼ if all per-headlight optics fail AND the server returned
         fallback="local" (HEADLIGHT_AI_FALLBACK_LOCAL=true), run the
         legacy canvas-only filter as last resort
```

Logs surface the path taken explicitly:
```
[Headlights] mode=full-image attempt=1
[Headlights] full-image attempt 1 validation { ok:false, reasons:[...], stats:{...} }
[Headlights] mode=full-image attempt=2 (stricter prompt)
[Headlights] mode=per-headlight (fallback)
```

## Provider abstraction

```
api/_lib/headlight/
  index.js              orchestrator (restoreHeadlights)
  prompts.js            DEFAULT_PROMPT + DEFAULT_NEGATIVE_PROMPT + STRENGTH_PRESETS
  crop.js               per-headlight crop math (computeLightCrop, transformLightToCrop)
  composite.js          PNG dimension guard + blend params
  mask.js               mask geometry helpers (pure JS, testable)
  providers/
    base.js             HeadlightRestorationProvider abstract class
    index.js            getProvider(env), resolveProviderConfig(env)
    openai.js           OpenAIHeadlightProvider
    replicate.js        ReplicateHeadlightProvider
    stability.js        StabilityHeadlightProvider
```

Each provider implements the same contract:

```js
class HeadlightRestorationProvider {
  async restoreHeadlightsWithAI({
    imageBase64, imageMime, maskBase64,
    prompt, negativePrompt, strength, size,
  }) { /* → { imageBase64, model, raw } */ }
}
```

Adding a new provider is a single file: extend the base class, register
it in `providers/index.js`, done.

## Environment variables

| Var                              | Required | Default   | Description |
|----------------------------------|----------|-----------|-------------|
| `HEADLIGHT_AI_ENABLED`           | no       | `true`    | Set to `false` to disable the feature server-side. The handler then refuses requests with a clear error. |
| `HEADLIGHT_AI_PROVIDER`          | no       | `openai`  | `openai` \| `replicate` \| `stability` |
| `HEADLIGHT_AI_API_KEY`           | yes\*    | —         | Provider API key. \*Optional if a provider-specific key is set: `OPENAI_API_KEY`, `REPLICATE_API_TOKEN`, `STABILITY_API_KEY`. |
| `HEADLIGHT_AI_MODEL`             | no       | per-provider | OpenAI: `gpt-image-1`. Replicate: `stability-ai/stable-diffusion-inpainting`. Stability: `sd3.5-large`. |
| `HEADLIGHT_AI_FALLBACK_LOCAL`    | no       | `false`   | When `true`, the API responds to AI failures with `fallback: "local"` and the client runs a **legacy canvas-only polish**. By default we surface the error and never pretend the AI succeeded. |

### Strength presets

| `strength` | `quality` | `input_fidelity` | When to use |
|---|---|---|---|
| `restore` *(default)* | medium | high | Per-headlight restoration. Conservative — preserves the original optic geometry. |
| `low`     | medium | high | Subtle change. |
| `medium`  | high   | high | Balanced. |
| `high`    | high   | low  | Aggressive — let the model diverge from the input. Use only if the optic is very oxidized and `restore` doesn't move enough. |

### Debugging

Append `?headlightDebug=1` to the URL. After a batch:

```js
// Per-headlight inspection in the devtools console:
window.__headlightDebug.lights.forEach((l, i) => {
  console.log(`Light #${i}`, l);
  document.body.appendChild(l.sentImage);   // crop sent to the AI
  document.body.appendChild(l.sentMask);    // mask sent to the AI
  if (l.aiResultRaw) {
    document.body.appendChild(l.aiResultRaw);     // raw AI output
    document.body.appendChild(l.aiResultAtCropSize); // resampled to crop size
    document.body.appendChild(l.beforeCrop);  // crop region before composite
    document.body.appendChild(l.afterCrop);   // crop region after composite
  }
  console.log(`Light #${i} meanDiff=${l.meanDiff?.toFixed(2)}`);
});
```

> 🔐 No API key is ever hardcoded. A `tests/headlight-secrets.test.js`
> assertion fails the build if anyone commits one in the headlight files.

## API contract

```
POST /api/lustrage-pro

{
  "imageBase64": "<base64, no data: prefix>",
  "imageMime":   "image/jpeg",
  "maskBase64":  "<base64 PNG, transparent = area to redraw>",
  "size":        "1024x768",
  "mode":        "ai",                 // only "ai" is supported
  "strength":    "low" | "medium" | "high",  // default: "medium"
  "prompt":         "...optional override...",
  "negativePrompt": "...optional override..."
}
```

**Success** (200):
```json
{
  "imageBase64": "...",
  "provider":    "openai",
  "model":       "gpt-image-1",
  "strength":    "medium",
  "blend":       { "expandOuter": 0.065, "expandInner": 0.035, "edgeBlur": 0.0025, "aiOpacity": 1.0 },
  "attempts":    [],
  "mode":        "ai"
}
```

**Failure** (4xx / 5xx):
```json
{
  "error":    "OpenAI image edit failed",
  "provider": "openai",
  "details":  { ... },
  "fallback": null,                   // or "local" if HEADLIGHT_AI_FALLBACK_LOCAL=true
  "blend":    null
}
```

## How the feature is triggered

In the SaaS UI:

1. Open a session and import photos.
2. In the **Setup** tab, toggle **Lustrage des optiques**
   (`headlightPolish` state in `src/App.jsx`). The toggle is gated by
   the user's subscription tier — confirmed via `canUseHeadlight`.
3. Run the batch (**Lancer le traitement**).

Per photo, `processPhoto()` calls
`aiPolishHeadlights(ctx, W, H, b64, { strength: "medium" })`, which
performs steps [2]–[7] above.

## Limitations / known issues

- The detector (api/headlights.js) is a GPT-4o-mini vision call; on
  unusual angles or partial views the polygon may be loose. The client
  filters lights with width or height < 1.5 % to drop spurious
  detections.
- gpt-image-1 sometimes returns a slightly smaller image (rounded to a
  multiple of 8). The orchestrator tolerates ±16 px before refusing.
- Replicate adds 5–30 s of inference latency. Use `medium` strength as
  default; `high` only when the lens is very oxidized.
- Stability AI requires the v2beta endpoint; if your account is
  v1-only, switch the provider.

## Adding a new provider

1. Create `api/_lib/headlight/providers/<name>.js`:
   ```js
   import { HeadlightRestorationProvider, ProviderRequestError } from './base.js';
   export class MyProvider extends HeadlightRestorationProvider {
     get name() { return 'mine'; }
     get defaultModel() { return 'mine-v1'; }
     async restoreHeadlightsWithAI(req) { /* ... */ }
   }
   ```
2. Register it in `api/_lib/headlight/providers/index.js`:
   ```js
   import { MyProvider } from './mine.js';
   const REGISTRY = { openai: ..., replicate: ..., stability: ..., mine: MyProvider };
   ```
3. (Optional) Add a provider-specific env-var fallback in
   `providerSpecificKey()`.
4. Add a test in `tests/headlight-providers.test.js`.

## Running locally

```bash
# Install deps
npm install

# Run the unit tests (no network, ~250 ms)
npm test

# Start the dev server (Vite)
npm run dev
# In another shell, start a Vercel-style API runtime, or test the
# endpoint with curl:
curl -X POST http://localhost:3000/api/lustrage-pro \
  -H "Content-Type: application/json" \
  -d '{
    "imageBase64": "...",
    "maskBase64":  "...",
    "size":        "1024x1024",
    "mode":        "ai",
    "strength":    "medium"
  }'
```

The tests use an in-process `MockProvider` injected via the `__registry`
hook in `providers/index.js`; no real API key is ever required to run
the suite.
